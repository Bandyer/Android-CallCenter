handleNotification

Unit handleNotification(String data, OnNotificationObserver onNotificationObserver)
Content copied to clipboard

Parses the notification payload provided as first argument. If the payload is in the expected format it will report a new call object, otherwise, if the payload cannot be parsed, or is malformed an error will be reported via the observer.

An error will be reported also in the case the call client is not connected, or another call is in progress. You should not pass the notification payload received from APNs as is. You must provide the string containing the data your back-end has received from the bandyer's back-end system.

Note: The call client must be connected to bandyer's platform before it can handle a remote notification. You must call this method after onCallClientInitialized(sessionUser) was called, otherwise it will fail even if the payload is valid.

Parameters

data

Bandyer's notification payload

OnNotificationObserver result listener
